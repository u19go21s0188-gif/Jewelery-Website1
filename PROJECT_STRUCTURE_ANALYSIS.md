# Project Structure & Analysis - Jewelery Website

## Project Overview
This is a **Full-Stack Web Application** built with:
- **Frontend**: React 19 + TypeScript + TanStack Router + TanStack Start
- **Backend**: Cloudflare Workers (Wrangler) + Node.js compatible runtime
- **Database**: Supabase (PostgreSQL)
- **Styling**: Tailwind CSS + Shadcn UI Components
- **Build Tool**: Vite
- **Package Manager**: Bun

## 📦 dist Folder Structure (Build Output)

The `dist/` folder contains the **compiled & optimized production build** generated by `npm run build`:

### dist/client/
**Purpose**: Frontend assets served to browsers
```
dist/client/
├── .assetsignore           # Asset exclusion rules
└── assets/                 # Bundled & minified assets
    ├── *.js               # Minified JavaScript chunks (route-based code splitting)
    ├── *.css              # Minified stylesheets
    ├── *.jpg              # Optimized images
    └── [Hash names]       # Content-hash filenames for cache busting
```

**Key Assets in dist/client/assets/**:
| File Pattern | Purpose |
|---|---|
| `admin-*.js` | Admin panel route chunks |
| `admin.categories-*.js` | Category management chunks |
| `admin.dashboard-*.js` | Dashboard analytics chunks |
| `admin.products-*.js` | Product management chunks |
| `admin.orders-*.js` | Order management chunks |
| `auth-*.js` | Authentication route |
| `category._slug-*.js` | Dynamic category pages |
| `index-*.js` | Home page |
| `createLucideIcon-*.js` | Lucide icon utility |
| `product-card-*.js` | Product card component |
| `styles-*.css` | Global CSS bundle |
| `hero-*.jpg` | Hero image (optimized) |
| Utility files | Helper function chunks |

### dist/server/
**Purpose**: Backend server code for Cloudflare Workers
```
dist/server/
├── index.js               # Main server entry point
├── .dev.vars             # Development environment variables
├── wrangler.json         # Cloudflare Workers config
├── .vite/                # Vite cache directory
└── assets/               # Server-side assets
    ├── *.js              # Server-side route handlers
    ├── start-*.js        # TanStack Start runtime
    ├── worker-entry-*.js # Cloudflare Worker entry point
    ├── _tanstack-start-manifest_*.js # Route manifest
    ├── __23tanstack-start-plugin-adapters_*.js # Adapter plugins
    └── [Other utilities]
```

**Key Server Files**:
| File Pattern | Purpose |
|---|---|
| `index.js` | Main server entry point |
| `start-*.js` | TanStack Start server runtime |
| `worker-entry-*.js` | Cloudflare Workers execution entry |
| `_tanstack-start-manifest_*.js` | Server-side route manifest |
| `admin-signup.functions-*.js` | Admin signup API |
| `router-*.js` | Server-side router |

    ↓

│   - Hash-based cache busting

## 🛠️ Development Workflow

1. **Local Dev**: Run `bun dev` → Starts Vite dev server with hot reload
2. **Build**: Run `bun build` → Creates optimized dist/ folder
3. **Preview**: Run `bun preview` → Test production build locally
4. **Deploy**: Upload dist/ to Cloudflare Workers or Vercel
5. **Lint**: Run `bun lint` → Check code quality
6. **Format**: Run `bun format` → Auto-format code

## 📝 Summary

This is a **modern, full-stack jewelry e-commerce platform** with:
- **Frontend** served as static assets from dist/client/
- **Backend** running on serverless Cloudflare Workers from dist/server/
- **Database** hosted on Supabase (PostgreSQL)
- **Optimized builds** with code splitting, CSS extraction, and image optimization
- **Production-ready** with proper caching, error handling, and monitoring setup

The dist folder is generated during build and contains everything needed for deployment to production servers.
